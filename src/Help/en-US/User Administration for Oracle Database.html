User Administration for Oracle Database

It is recommended that each user be assigned a personal database account with only the required privileges. The pdfkeeper database account should only be used for pdfkeeper schema administration, not for general use. For more information on Managing Users, please refer to the Oracle Database Documentation.

Perform the following steps for any user that will need access to the pdfkeeper schema:

  1. From a Command Prompt, execute sqlplus

  2. When logged into the database server computer using the account that installed Oracle Database, enter connect / as sysdba. Otherwise, enter the database connect string in the format username/password@host:port/service_name where username is a database account that must be a member of SYSDBA (SYSTEM is a member of SYSDBA); host is the host name or IP address of the database server computer (use localhost or 127.0.0.1 if logged onto the database server computer); port is the listening port number on the database server (the default is 1521); and service_name is the service name of the database instance or pluggable database to access (for Oracle Database Express Edition, the default is XE). For Oracle Database Express Edition, it is recommended to use the default pluggable database XEPDB1.

  3. For a new user, enter create user user_name default tablespace users identified by "password";

  4. Enter grant create session to user_name;

  5. Enter quit

  6. From a Command Prompt, execute sqlplus

  7. When logged into the database server computer using the account that installed Oracle Database, enter connect pdfkeeper. Otherwise, enter the database connect string in the format pdfkeeper/password@host:port/service_name where host is the host name or IP address of the database server computer (use localhost or 127.0.0.1 if logged onto the database server computer); port is the listening port number on the database server (the default is 1521); and service_name is the service name of the database instance or pluggable database to access (for Oracle Database Express Edition, the default is XE). For Oracle Database Express Edition, it is recommended to use the default pluggable database XEPDB1.

  8. For a new account, enter the following statement to grant the user select/query access: grant select on docs to user_name;

  9. To give the user the ability to upload documents, enter the following statements: grant insert on docs to user_name; and grant select on docs_seq to user_name;

  10. To give the user the ability to edit notes, assign a category, and set the follow-up state, enter the following statement: grant update on docs to user_name;

  11. To give the user the ability to delete documents, enter the following statement: grant delete on docs to user_name;

  12. To prevent the user from uploading documents, enter the following statements: revoke insert on docs from user_name; and revoke select on docs_seq from user_name;

  13. To prevent the user from editing notes, assigning a category, and setting the follow-up state, enter the following statement: revoke update on docs from user_name;

  14. To prevent the user from deleting documents, enter the following statement: revoke delete on docs from user_name;
